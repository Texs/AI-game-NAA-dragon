# Contributing to Mathematical Operations Library

Thank you for considering contributing to our project! We welcome contributions from everyone, regardless of experience level.

## Getting Started

1. Fork the repository
2. Create a new branch for your feature or bug fix
3. Make your changes
4. Add tests if applicable
5. Run the test suite to ensure everything passes
6. Submit a pull request

## Setup Instructions

To get started with the project, follow these steps:

### Prerequisites
- Node.js (v14 or higher)
- Python (v3.6 or higher)

### Installation
1. Clone the repository:
   ```bash
   git clone https://github.com/your-username/repository-name.git
   ```

2. Navigate to the project directory:
   ```bash
   cd repository-name
   ```

3. Install JavaScript dependencies (if any):
   ```bash
   npm install
   ```

## Running Tests

### JavaScript Tests
```bash
npm test
```

### Python Tests
```bash
python test.py
```

Or using npm script:
```bash
npm run test:py
```

## Code Style

### JavaScript
- Use 4-space indentation
- Use single quotes for strings
- End statements with semicolons
- Follow JSDoc conventions for documentation

### Python
- Follow PEP 8 style guide
- Use docstrings for functions and modules
- Use descriptive variable names

## Pull Request Guidelines

1. Keep pull requests small and focused on a single issue or feature
2. Provide a clear description of the changes
3. Include tests for new functionality
4. Update documentation as needed
5. Ensure all tests pass before submitting

## Reporting Issues

When reporting issues, please include:
- A clear description of the problem
- Steps to reproduce the issue
- Expected vs. actual behavior
- Any relevant error messages
- Environment information (OS, version, etc.)

## Development Workflow

1. Create an issue describing the feature or bug fix
2. Discuss with maintainers if needed
3. Create a branch named appropriately (e.g., `feature/new-functionality` or `bugfix/issue-description`)
4. Implement the changes
5. Add/update tests
6. Update documentation if necessary
7. Submit a pull request referencing the original issue

## Questions?

If you have questions about contributing, feel free to open an issue or contact the maintainers.

Happy coding!

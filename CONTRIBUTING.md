Thank you for your interest in contributing to the AI-game-NAA-dragon project! We appreciate your time and effort in making this project better.

## Getting Started

1. Fork the repository
2. Create a new branch for your feature or bug fix
3. Make your changes
4. Add tests if applicable
5. Run the test suite to ensure everything passes
6. Submit a pull request

To run JavaScript tests:

To run Python tests:
```bash

```

## Code Style
## Code Style Guidelines

### JavaScript
- Use 4-space indentation
- Use single quotes for strings
- End statements with semicolons
- Follow JSDoc conventions for documentation

### Python
- Follow PEP 8 style guide
- Use docstrings for functions and modules
- Use descriptive variable names

## Pull Request Guidelines

1. Keep pull requests small and focused on a single issue or feature
2. Provide a clear description of the changes
3. Include tests for new functionality
4. Update documentation as needed
5. Ensure all tests pass before submitting
- Use semicolons at the end of statements
- Write JSDoc comments for all functions
- Follow the ESLint configuration provided in `.eslintrc.json`

### Python
- Follow PEP 8 style guide
- Use docstrings for all functions
- Use descriptive variable names
- Keep functions focused and small

## Project Structure

- `app.js` - Main JavaScript application logic
- `app.py` - Main Python application logic
- `test.js` - JavaScript unit tests
- `test.py` - Python unit tests
- `index.html` - Main HTML page for the CI/CD project
- `package.json` - Node.js package configuration
- `.eslintrc.json` - ESLint configuration

## Pull Request Process

1. Ensure your code follows the style guidelines
2. Update documentation if necessary
3. Add tests for new functionality
4. Make sure all tests pass before submitting
5. Provide a clear description of your changes in the pull request

## Reporting Issues

When reporting issues, please include:
- A clear description of the problem
- Steps to reproduce the issue
- Expected vs. actual behavior
- Any relevant error messages
- Environment information (OS, version, etc.)

## Development Workflow

1. Create an issue describing the feature or bug fix
2. Discuss with maintainers if needed
3. Create a branch named appropriately (e.g., `feature/new-functionality` or `bugfix/issue-description`)
4. Implement the changes
5. Add/update tests
6. Update documentation if necessary
7. Submit a pull request referencing the original issue
